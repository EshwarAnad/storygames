\nBuilding on Thanuir\u2019s ideas here\u2026\n

You\u2019re gonna need two things.

  1. A fantasy sandbox. Full of rando tables as per usual. The more big it is the better for purps of diluting the custom stuff so that not break the \u201cno paper after seeing rock\u201d principle too much. For the purps of this, I want there to be a chance that something is never ever rolled.
  2. A list of traits, ideals, bonds, flaws and trinkets.
CategoryMandatory for players to addMandatory for DM to use
Personality Trait0, it\u2019s just good to have some0, but if you find an actionable one go for it
Ideal11
Bonds3 (two to other players and one normal one)1 if they provided an normal one, you gotta use it
Flaws0, it\u2019s just good to have one0, but if you find an actionable one go for it
Trinket11
Background specific stuff such as hermit discovery0, this is optional\u221e, DMs, if this is there, this needs to be used
XGE class specific stuff0, this is optional\u221e, DMs, if this is there, this needs to be used
XGE life path stuff, friends & family rolled up0, this is optional\u221e, DMs, if this is there, this needs to be used
Trinkets fell away because they never got reincorporated into our play since it\u2019s so blorby.

But here they\u2019re gonna get used again.

Players, you need to hand in an ideal, a trinket, and three bonds. XP is frozen until you comply. Traits & flaws are good for you. One of the bond is your normal background bond, the other two bonds are Hillfolk \u201cunattainable wants\u201d to other PCs.

DM, you\u2019re gonna make a big old list. The \u201cmaster challenge list\u201d.

To that list you add all the ideals, bonds, trinkets, background-specific stuff, XGE-stuff, and if you find any exceptionally actionable flaws & traits [you\u2019ll gonna know what that means later in this process] add them in too. Make some sorta note which character they came from.\n

When prepping

For each item on the \u201cmaster challenge list\u201d you make sure that there is at least one relevant thing on your other tables that are in use. Could be encounter tables, rando NPC quality tables, treasure drop tables etc. If there isn\u2019t, you add one.

An optional, but good prompt for you when you have your prepping hat on is \u201chow can I make this go away?\u201d A promise goes away by fulfilling it or breaking it, a key goes away by finding the lock for it or having it stolen, an ideal goes away by being given up on etc. Careful now; a lot of posts on /r/rpghorrorstories are about \u201cThe DM took away the thing about my character that was cool\u201d. So IDK.

So a promise to rescue your father can lead to the actual father being locked up somewhere. Idk.

The items in the \u201cmaster challenge list\u201d cause other things to be added to your normal rando tables. Again, stick to tables that are in active use in the particular campaign type that you\u2019re running. (Maybe you can add a new category of table and start using it more, such as an \u201cNPC relevancy\u201d table to add onto any other NPC they meet, buuut\u2026 an entry such as \u201cThis is Alice\u2019s father\u201d might end up feeling super post-hoc and un-blorby and retconny and fake. TODO solve this problem before implementing the rule.)

When a PC dies, remove their stuff from the \u201cmaster challenge list\u201d but not the stuff that was added to the normal rando table. So Alice\u2019s father is still running around out there.

Always keep the master challenge list current and when prepping always make sure that each entry on the master challenge list has at least one representative on the normal rando tables.\n

When running

You need to absolutely forget why you put the thing on the list. No longer is it\u2019s purpose to \u201cremove the trait\u201d or whatever. It\u2019s just a normal encounter with Alice\u2019s father or a treasure chest that matches Bob\u2019s trinket key or whatever. TODO this type of compartmentalization between prepping&running hats is difficult, not sure how to fix\u2026?

What\u2019s cool though is that you don\u2019t need to refer to the master challenge list constantly fretting \u201cis there something relevant?\u201d The items on your lists are doing the work for you, just run the game normally & blorbily. Let the meetings with these entities play out naturally.\n

IDK this can sometimes come across as really fake

We had this problem in our CoS campaign where there actually was an old ship\u2019s flag on the rando treasure drop table and our players though this was fakily inserted to match one of the PC\u2019s sailor background. Which, uh, would\u2019ve been kind of a toothless insert not really pulling the weight of undermining the feeling of tangibility that much.

So some kinda transparency of method about this is prob a good idea. IDK. Gonna think some more about this before I try to implement it.

    \n # Unique things

    Place them. Add to relevant rumour and random encounter tables and what not.

    Lazy option: Add them to the generic encounter/loot table.

    # Frequent things

    Someone really likes cats, has a controversial political opinion, etc.

    These should be guaranteed to happen fairly often. This is difficult, because one needs a process for it and adding new processes is challenging, while modifying existing ones is, too.

    # Rare things

    Add to the relevant random table. Maybe place a few if there are good and relevant places.
